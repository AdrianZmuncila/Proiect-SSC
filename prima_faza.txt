🎯 Rolul fiecărei componente în proiect

Proiectul tău simulează un mic mediu de rețea în care vrei să detectezi un posibil atac Man-in-the-Middle (MITM).

Ai două mașini virtuale:

Kali Linux

Ubuntu Server

🟢 1️⃣ Ubuntu Server – Rolul lui

Ubuntu Server reprezintă:

✔ un sistem legitim din rețea
✔ o resursă care comunică
✔ potențială țintă a unui atac MITM

Ce face concret:

Are IP: 192.168.100.10

Trimite trafic (ping către Kali)

Răspunde la solicitări ARP

Poate fi folosit pentru a simula o anomalie (schimbare MAC)

🔎 Cum explici profesoarei

Ubuntu Server reprezintă un nod legitim din rețea, care generează trafic normal. Este folosit pentru a observa comportamentul ARP în condiții normale și pentru a simula o modificare IP-MAC în scopul testării mecanismului de detecție.

🔵 2️⃣ Kali Linux – Rolul lui

Kali nu este atacatorul în proiectul tău.
Kali este sistemul de monitorizare.

Are rolul de:

✔ a captura pachetele ARP
✔ a analiza relația IP → MAC
✔ a detecta schimbări suspecte
✔ a genera alerte

Pe Kali rulează scriptul Python care:

ascultă traficul ARP

ține o tabelă IP → MAC

compară dacă mapping-ul se schimbă

anunță dacă apare o anomalie

🔎 Cum explici profesoarei

Kali Linux este utilizat pentru monitorizarea traficului din rețea. Scriptul implementat analizează pachetele ARP și detectează modificări suspecte ale asocierii IP-MAC, indicator specific unui posibil atac MITM.

🧠 3️⃣ Ce înseamnă ce ai văzut în tcpdump

Când ai rulat:

sudo tcpdump -i eth0 arp

Ai văzut:

ARP, Request who-has 192.168.100.10 tell 192.168.100.20

Înseamnă:

👉 Kali întreabă:
„Cine are IP-ul 192.168.100.10?”

Serverul răspunde:

ARP, Reply 192.168.100.10 is-at 08:00:27:51:17:a7

Înseamnă:

👉 „IP-ul 192.168.100.10 este asociat cu MAC-ul 08:00:27:51:17:a7”

Acesta este comportament normal.

🔥 Ce înseamnă schimbarea MAC-ului

Când tu modifici MAC-ul pe server:

IP-ul rămâne același

dar MAC-ul se schimbă

Scriptul detectează:

„Acest IP avea alt MAC înainte. De ce s-a schimbat?”

Exact asta face un atac MITM prin ARP spoofing:

atacatorul trimite răspunsuri ARP false

spune că IP-ul serverului are alt MAC (al atacatorului)

Deci scriptul tău caută fix acest simptom.

🎓 Explicația simplă pentru prezentare

Poți spune așa:

În mod normal, fiecare adresă IP din rețea este asociată cu un singur MAC address. Scriptul monitorizează aceste asocieri. Dacă pentru același IP apare un MAC diferit, sistemul generează o alertă, deoarece acesta este un comportament caracteristic atacurilor de tip ARP spoofing, utilizate în scenarii MITM.

Adresa MAC funcționează la:

Layer 2 – Data Link (model OSI)

Rolul ei principal este:

👉 să permită comunicarea între dispozitive în aceeași rețea locală (LAN)

🔹 Cum funcționează practic

Când un calculator vrea să trimită date către altul în aceeași rețea:

1️⃣ Știe IP-ul țintei
2️⃣ Folosește ARP (Address Resolution Protocol)
3️⃣ ARP află ce MAC corespunde acelui IP
4️⃣ Datele sunt trimise la acea adresă MAC🔥 Rolul adresei MAC într-o rețea
